## Feedback Guidelines

### For Bug Reports
1. **Describe the problem**: What happened vs. what you expected
2. **Steps to reproduce**: Detailed steps to recreate the issue
3. **Environment**: OS, Python version, package versions
4. **Code sample**: Minimal example that demonstrates the issue
5. **Error messages**: Include full error messages and stack traces

### For Feature Requests
1. **Use case**: Describe what you're trying to achieve
2. **Current limitations**: What's blocking you now?
3. **Proposed solution**: Your idea for how to solve it
4. **Alternatives**: Other approaches you've considered
5. **Impact**: How would this help you and others?

### For Documentation
1. **Specific location**: Which page or section needs improvement?
2. **What's unclear**: What information is missing or confusing?
3. **Suggested improvement**: How would you improve it?
4. **Your background**: What's your experience level?

## Response Times

We aim to respond to feedback within:
- **Bug reports**: 2-3 business days
- **Feature requests**: 1 week
- **Documentation issues**: 3-5 business days
- **General discussions**: 1-2 weeks

## Priority Levels

### High Priority
- **Security vulnerabilities**: Immediate attention
- **Critical bugs**: Blocking core functionality
- **Breaking changes**: Issues that break existing code

### Medium Priority
- **Feature requests**: Useful new functionality
- **Performance issues**: Optimization opportunities
- **Documentation gaps**: Missing or unclear information

### Low Priority
- **Nice-to-have features**: Convenience improvements
- **Minor bugs**: Non-blocking issues
- **Enhancements**: General improvements

## Getting Started

### 1. Set Up Development Environment

```bash
# Clone the repository
git clone https://github.com/fireworks-ai/eval-protocol.git
cd eval-protocol

# Create a virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -e ".[dev]"

# Install pre-commit hooks
pre-commit install
```

### 2. Run Tests

```bash
# Run the test suite
pytest

# Run tests with coverage
pytest --cov=eval_protocol

# Run linting
flake8 eval_protocol/
black --check eval_protocol/
```

### 3. Make Your Changes

1. **Create a branch**: `git checkout -b feature/your-feature-name`
2. **Make your changes**: Follow our coding standards
3. **Add tests**: Ensure your changes are well-tested
4. **Update documentation**: Keep docs in sync with code changes
5. **Run tests**: Make sure everything still works

### 4. Submit a Pull Request

1. **Push your branch**: `git push origin feature/your-feature-name`
2. **Create a PR**: Use our pull request template
3. **Address feedback**: Respond to code review comments
4. **Get merged**: Celebrate your contribution! üéâ

## Development Guidelines

### Code Style
- **Python**: Follow PEP 8, use Black for formatting
- **Type hints**: Use type hints for all public APIs
- **Docstrings**: Use Google-style docstrings
- **Comments**: Focus on "why" rather than "what"

### Testing
- **Unit tests**: Test individual functions and classes
- **Integration tests**: Test end-to-end workflows
- **Coverage**: Aim for >90% test coverage
- **Edge cases**: Test error conditions and edge cases

### Documentation
- **API docs**: Auto-generated from docstrings
- **User guides**: Step-by-step instructions
- **Examples**: Runnable code examples
- **Tutorials**: Learning-oriented content

## Reward Function Guidelines

When contributing reward functions:

### Quality Standards
- **Clear purpose**: Document what the function evaluates
- **Robust implementation**: Handle edge cases gracefully
- **Performance**: Consider efficiency for large datasets
- **Reproducibility**: Ensure consistent results

### Example Structure
```python

def your_reward_function(query: str, response: str) -> float:
    """
    Brief description of what this function evaluates.
    
    Args:
        query: The input query or prompt

    Returns:
        Score between 0.0 and 1.0, where 1.0 is best
        
    Example:
        >>> score = your_reward_function("What is 2+2?", "4")
        >>> print(score)  # 1.0
    """
    # Implementation here
    pass
```

### Documentation Requirements
- **README**: Explain purpose and usage
- **Examples**: Provide usage examples
- **Benchmarks**: Include performance data if relevant
- **Dependencies**: List any additional requirements

## Community Standards

### Code of Conduct
All contributors must follow our [Code of Conduct](https://github.com/fireworks-ai/eval-protocol/blob/main/CODE_OF_CONDUCT.md).

### Communication
- **Be respectful**: Treat everyone with kindness and respect
- **Be constructive**: Provide helpful feedback and suggestions
- **Be patient**: Remember that everyone is learning
- **Be inclusive**: Welcome newcomers and different perspectives

### Review Process
- **Maintainer review**: All PRs require maintainer approval
- **Automated checks**: CI/CD must pass before merge
- **Community feedback**: Encourage community participation in reviews
- **Documentation**: Changes must include relevant documentation updates

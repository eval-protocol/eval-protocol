title: Starting the UI
---

Reviewing model outputs by hand is a key part of evaluating quality. EP makes
this process simple by offering an easy-to-use, locally hosted UI you can set up
in minutes.

To start the UI, simply run the following command and open your browser to [http://localhost:8000](http://localhost:8000).

```bash CLI focus={1,11}
> ep logs
INFO:eval_protocol.utils.vite_server:Initialized Vite server for build directory: /Users/dylanhuang/Git/python-sdk/vite-app/dist
INFO:eval_protocol.utils.logs_server:LogsServer initialized on localhost:8000
INFO:eval_protocol.utils.logs_server:Starting LogsServer on localhost:8000
INFO:eval_protocol.utils.logs_server:Serving files from: /Users/dylanhuang/Git/python-sdk/vite-app/dist
INFO:eval_protocol.utils.logs_server:WebSocket endpoint available at /ws
INFO:eval_protocol.utils.logs_server:Evaluation watcher started
INFO:     Started server process [35036]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://localhost:8000 (Press CTRL+C to quit)
```

<Note>
Ensure the `eval-protocol` package is installed so the `ep` CLI is available.

<Tabs>
  <Tab title="uv">

```bash
# Add to your project (installs the `ep` CLI in your environment)
uv add eval-protocol

# Verify installation
uv run ep --version
```

  </Tab>
  <Tab title="pip">

```bash
# Install or upgrade
pip install -U eval-protocol

# Verify installation
ep --version
```

  </Tab>
</Tabs>
</Note>

Once you navigate to the UI at http://localhost:8000, you will see a table of
evaluation rows that you can click to inspect.

<Frame caption="First look at the UI">
![Table View](/assets/table.png)
</Frame>

Whenever you run an `@evaluation_test`—whether from the VSCode Test
Explorer/Debugger or from the CLI via `pytest`—the UI automatically shows
`running` tests and you can watch rollouts live in the chat interface. When a
test finishes, detailed evaluation results appear to the right of the chat.
Tests are stored under a SQLite database on your local device at
`.eval_protocol/logs.db` in the root of your Python project.

<Tabs>
  <Tab title="Run with uv">

```bash
# Run your tests (UI will reflect live rollouts)
uv run pytest
```

  </Tab>
  <Tab title="Run with pip">

```bash
# Run your tests (UI will reflect live rollouts)
pytest
```

  </Tab>
</Tabs>
